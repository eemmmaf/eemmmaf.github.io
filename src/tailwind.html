Tailwind

Jag har testat på ramverket Tailwind. Tailwind gör det möjligt att snabbt bygga en modern webbplats, utan att behöva skapa css-kod utanför HTML-filen. Tailwinds skapare, Adam Wathan, skriver på ramverkets webbplats att hans kod har gått från att skrivas väldigt semantiskt till att skrivas funktionellt, "functional CSS". Med Tailwind skrivs klasserna direkt i HTML-koden och det finns förutbestämda klasser som går att använda, istället för att skapa klasserna själv i en CSS-fil.


Länk till ramverkets webbplats

Fördelar enligt Tailwind

  • - Tid sparas eftersom att användaren inte behöver fundera ut klassnamn.
  • - Vid traditionellt skapad CSS blir ens CSS-fil större varje gång en klass läggs till i dokumentet. Med Tailwind är allting återanvändbart.
  • - Förändringar känns säkrare. Vid traditionellt skapad CSS är förändringarna i CSS-filerna globala och det kan vara svårt att veta vad som förändras om något ändras där. Tailwind-klasserna i HTML-dokumentet är lokala och vid en ändring ändras endast det elementet.
  • - Till skillnad mot att använda inline-style går det att använda pseudo-klasser, media quiries, pseudo-element och attribut-selektorer på HTML-elementen.
  • - Det blir allt vanligare att skapa en style för dark-mode. Tailwind har gjort det enkelt att skapa en mörkare version med hjälp av att använda "dark".
  • - Vid installering av Tailwind följer en config-fil med. I den går det att anpassa sitt tema, exempelvis skapa ett färgtema eller brytpunkter. Det går också att lägga till plugins i den.

Min reflektion

Fördelar

  • Sparar tid genom att klasserna redan finns och det inte behövs skrivas någon CSS-kod.
  • Väldigt anpassningsbart. Det går att anpassa och skapa sitt eget tema i tailwind-config-filen. Det kändes väldigt användbart att kunna skapa sitt egna färgtema.
  • Slipper hoppa mellan HTML och CSS-filer för att skriva CSS-kod, eftersom att allt skrivs i HTML-dokumentet.
  • Ingen CSS-kod skapas i onödan. När jag skapar CSS-kod på det traditionella sättet kan ibland klasser skapas i onödan och ta upp plats. Med Tailwind märker jag direkt om jag behöver stylen eller inte.
  • "Mobile-first"-standard

Nackdelar

  • Många nya klasser att lära sig. Det tar ett tag att lära sig de nya klasserna, även om jag anser att jag har bra kunskap om traditionell CSS.
  • Lite krångligt att installera och konfigurera, till skillnad mot att använda CDN. Tailwind har dock väldigt bra dokumentation kring hur det ska installeras, så det går snabbt att komma igång om man är van vid liknande installationer.
  • Vissa komponenter saknas. Jag var tvungen att använda ett plugin för formulär för att kunna styla formulär.
  • Inte det bästa alternativet för personer som inte har erfarenhet av CSS. Jämfört med Materialize krävdes det mer läsning i dokumentationen för att förstå hur ramverket fungerar.
